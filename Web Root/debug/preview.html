Social Media Preview Test

Pages without actual IMG tags don't have previews on social media sites (Facebook and Linkedin are the big ones). Most of this side uses background images. Testsing to see if "display: none" or html5 hidden works best for social media preview. Facebook preview is controlled by a PHP file (safe_image.php). The safe image is 1000 x 521 pixels. I can't see the file or what it's doing, but the logic seems to be putting the IMG in the HTML file, then hiding it with CSS. Guessing the PHP is to dynamically inject the image when the article appears on Facebook. If all goes according to plan, at least of the preview images will show up on Facebook and Linkedin!
